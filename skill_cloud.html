The CrimeMapping.com website allows public safety agencies provide recent crime information to the public. The site has over 450 agencies, including San Diego PD, throughout North America publishing data. The ASP.NET MVC site is deployed in Azure.

The development, staging and production environments are all within Azure as different subscriptions. This allows for the internal billing to be easily organized and makes for a great developer experience.

Technology
  • C#
  • JavaScript
  • ArcGIS JavaScript API
  • Azure SQL
  • Web Jobs
  • Redis
  • StackExchange.Redis

High Availability

The website is deployed into two Azure data centers and there is an Azure Traffic Manager used to distribute the traffic. The setup of the data centers is the same; there is a Web App that hosts the website, Redis for secondary caching and Azure SQL.

Geo-replication is enabled with Azure SQL with the West Coast data center being the master.

Automation

The public have the ability to sign up for Alerts that provide email notification when there is a new crime found within the spatial filter provided. Every day, the Azure Scheduler calls a Web Job that gets the active alerts found within the database and then loads these alerts unique into the queue.

There is a continuous Web Job running that reads the alerts from the queue and does the processing. These Web Jobs are running in both data centers and are configured to query the local instance of Azure SQL. When new crime records are found, an email notification is sent. The application is configured to use SendGrid or AWS Simple Email Service (SES). Every month, there are 2.2 million emails sent.
